Le readiness (prospectif) répond « peux-tu pousser aujourd'hui ? » d'après ton sommeil, tes courbatures, ta forme (charge chronique − aiguë) et ton ACWR. Récupération = rétrospectif ; progression = tendance de tes records (e1RM). Tout reste indicatif tant que les entrées manquent.

    


    
    

      
Pilotage de la charge

      

        

        

      

      

        

        

      

      

        
ACWR
—

        
Risque
—

        
Assiduité
—

      

      

      

      
Charge = tonnage de séance (Σ charge×reps). Aiguë (7 j) vs chronique (28 j) : l'écart qui se creuse (ACWR > 1,5) signale un pic à risque ; 0,8–1,3 = zone optimale. Tout se fiabilise à mesure que tu accumules des séances.

    


    

      
Volume par groupe musculaire

      

      
Séries de travail des 7 derniers jours par groupe (déduit du nom d'exercice). Repère hypertrophie : 10–20 séries/sem/groupe. Les écarts marqués sont signalés ; les groupes non reconnus sont ignorés.

    


    

      
Cycles d'entraînement

      

      

      
Périodisation réelle déduite de ta charge hebdo : accumulation (charge qui monte), deload (semaine d'allègement), maintien. Un cycle accumulation → deload favorise la surcompensation.

    


    

      
Corrélations

      

      

      
Croise tes données pour révéler tes leviers (sommeil, volume, kcal). Indicatif : une corrélation n'est pas une causalité, et se fiabilise avec plus de relevés.

    Séance du jour
    
      
    
        
        
      
    
      
      
    
      
    
      
    Chaque exercice affiche son objectif « 0 doute » : suis-le, et la barre de niveau te dit quand monter la charge. Active « Unilatéral · charge d'un côté » si tu travailles bras/jambe séparément : la charge saisie est alors celle d'un seul côté (le volume compte les deux). Une rep avec élan ne compte pas.
    
    
    

    
    

    
    
      
    Progression par exercice
    
      
    
        
        
      
    
      
    
      
    
      
    Courbe du 1RM estimé (Epley, sur ta meilleure série) et du volume total. La pente vers le haut = surcharge progressive réussie. La projection extrapole ton rythme récent vers ta prochaine marche (+2,5 / +5 kg).

    Synchro automatique (Gist GitHub)
    
      
    Sauvegarde et synchro PC ↔ mobile automatiques via un Gist GitHub secret. À chaque modif, le carnet l'écrit dans le gist ; au démarrage il le relit et fusionne (par date / id). Crée un token fine-grained avec le seul scope « Gists » sur GitHub, colle-le ci-dessous, laisse l'ID vide pour qu'un gist soit créé automatiquement.
    
      
      
    Synchro non configurée.
    
      
    
        
    
          
          
        
    
        
    
          
          
        
    
        
        
    
          
          
        
    
      
    
      
    🔒 Le token reste uniquement dans ce navigateur (saisie + stockage local). Il n'est jamais dans le code, le JSON exporté, ni le gist de données. Le dépôt GitHub Pages peut rester public sans risque : aucun secret n'est dans le code. La synchro échoue silencieusement hors-ligne — tes données restent en local et repartent au retour du réseau.
    
    
    

    
    
      
    Export / import manuel (filet de sécurité)
    
      
    Les données sont enregistrées automatiquement dans ce navigateur. Le JSON est le filet de secours si tu n'utilises pas la synchro Gist : exporte ici, importe là-bas. L'export détaille tout : pesées, mensurations, repas réellement mangés (avec quantités), séances (charge × reps par série), programmes et liste de courses. L'import fusionne intelligemment (par date / id, le fichier importé gagne).
    
      
    Pour importer un programme seul, le fichier JSON peut ne contenir que la clé programmes : [{"id","nom","jours":[{"id","nom","exercices":[{"nom","series","reps","note"}]}]}].
    
      
      
    
      
      
      
      
    
      
    L'historique des séances de muscu n'est jamais réinitialisé (seuls les repas se décochent à minuit). Une copie miroir est conservée en local au cas où le navigateur purgerait le stockage.
    
    
    
    
    
      
    Zone rouge
    
      
      
    Irréversible. Exporte d'abord. Efface aussi la copie cloud (Gist) si la synchro est active — le token, lui, reste (utilise « Oublier le token » pour l'enlever).
